# gold-bot

Бот для внутридневной торговли XAUUSD (золото) через брокера XM и терминал MetaTrader 5.
Проект ведётся поэтапно: подключение -> данные -> исследование -> бэктест -> демо -> live.
Подробные правила и риск-менеджмент - в `config.py`.

**Текущий этап: 3 - бэктест.**

## Итоги этапа 2 (закрыт, 10.07.2026)

Гипотеза подтвердилась: окно **17:00-21:00 (Лондон+Нью-Йорк) - самое волатильное
непрерывное окно суток**, торгуем в нём (см. `TRADING_WINDOW_START/END` в `config.py`).

Средний часовой размах GOLD (без искажающих часов, см. ниже):
- Торговое окно 17:00-21:00: **$8.41/час**
- Середина дня 11:00-15:00:  $5.14/час
- Весь день в среднем:       $5.34/час
- Окно волатильнее середины дня на **64%**. Топ-5 часов: 18, 17, 19, 20, 6.

Важная находка: час **2:00 (UTC+5)** показывал аномально высокий размах ($13.33
среднее) - разбивка по дням недели показала, что это стабильно повторяется
**каждый будний день** (Пн-Пт ~$13-14, а Сб/Вс - почти ноль). Это НЕ недельный
гэп, а технический артефакт: 2:00 по нашему времени = полночь по серверу XM
(UTC+3) - момент дневного ролловера (начисление свопов), когда спред резко
расширяется и котировки "рвутся". Реального движения там нет и торговать это
нельзя - час исключён из статистики автоматически (`research/hourly_volatility.py`
сам это детектит и объясняет в выводе).

- Изначально MT5 отдавал только последние ~100 000 баров на M5/M15 (терминал по

  `time_utc` (настоящее UTC) и `time_local` (UTC+5, твоё время) - на этапе 2
  используем `time_local`.
- Файлы `data/*.parquet` в git не коммитятся (см. `.gitignore`) - они легко
  перекачиваются заново скриптом `bot/fetch_history.py`.

## Этап 0: установка и проверка подключения

Выполняется **на Windows**, где установлен терминал MetaTrader 5.
Библиотека `MetaTrader5` умеет работать только на Windows - она общается с уже
запущенным терминалом напрямую, поэтому со стороны Linux/сервера это не запустить.

### 1. Установите терминал и откройте демо-счёт XM

1. Скачайте и установите MetaTrader 5 с сайта XM (или через "Personal Area" на xm.com).
2. Откройте демо-счёт (Demo Account) в самом терминале: `Файл -> Открыть счёт -> XM ...`,
   выберите демо, укажите депозит (например, 1000 USD).
3. Убедитесь, что вы залогинены и видите котировки в Market Watch. Добавьте золото,
   если его нет в списке (правый клик по списку -> Show All или Symbols). Точное имя
   символа отличается у разных account type XM - на нашем счёте это `GOLD`,
   `check_connection.py` сам подскажет имя, если оно не совпадёт с `config.py`.
4. Включите автоторговлю: кнопка "Algo Trading" на верхней панели должна быть зелёной.

### 2. Установите Python-зависимости

В терминале Windows (PowerShell или cmd), в папке `gold-bot`:

```powershell
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

### 3. Запустите проверку подключения

```powershell
python bot\check_connection.py
```

Скрипт должен вывести:
- баланс и эквити демо-счёта;
- текущие bid/ask по золоту;
- фактический спред в пунктах и в долларах на лот 0.01;
- время последнего тика и реальное UTC-время (для расчёта смещения сервера XM от UTC).

Если что-то не подключается - в самом скрипте есть подсказки по частым причинам
(терминал не запущен, автоторговля выключена, неверное название символа и т.п.).

### 4. После успешного запуска

1. Впишите разницу между "временем тика" и "реальным UTC" в `config.py` ->
   `SERVER_UTC_OFFSET_HOURS`. Она понадобится на этапе 2 для анализа по часам суток.
2. Сообщите мне баланс демо-счёта, спред и фактическую разницу времени - зафиксируем
   этап 0 закрытым и перейдём к этапу 1 (выгрузка истории XAUUSD в parquet).

## Правила проекта (не нарушать)

- Работаем строго по этапам, не перепрыгиваем вперёд.
- Никогда не подключаемся к реальному счёту без явного письменного подтверждения в чате.
- Каждая позиция открывается только со стоп-лоссом, риск на сделку не выше 1% депозита.
- Подробности риск-менеджмента - в `config.py`.
