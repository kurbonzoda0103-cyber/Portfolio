# gold-bot (проект переведён на Bybit / портфель монет)

Бот для алготрейдинга **на Bybit, портфель топ-N USDT-перпетуалов по объёму**
(не одна монета — независимые позиции сразу по нескольким). Название папки
осталось историческим (`gold-bot`) — проект начинался с золота через
MetaTrader5/XM, но 10.07.2026 полностью переведён на Bybit; MT5 больше не
используется вообще. Старый прогресс по золоту/EURUSD остался только в
git-истории этой ветки, в рабочем коде его больше нет.

Проект ведётся поэтапно: подключение -> данные -> исследование -> бэктест ->
демо -> live. Жёсткие риск-правила зашиты отдельно в `risk_gate.py` — ни
стратегия, ни бэктест, ни живой бот не могут их обойти напрямую.

**Текущий этап: 0 - подключение.** (нумерация этапов начата заново под Bybit)

## Почему pybit, а не ccxt

Официальный SDK Bybit. Ключевая причина выбора: нужна честная поддержка
**Demo Trading** (виртуальный баланс на РЕАЛЬНЫХ рыночных данных mainnet, а
не пустой testnet с нереалистичными ценами и почти нулевой ликвидностью) — у
pybit это отдельный явный флаг (`demo=True`), у ccxt такой точной поддержки
именно Demo Trading Bybit нет. Раз проект живёт только на одной бирже, не
нужна и мультибиржевая абстракция ccxt — она добавила бы лишний слой между
кодом и конкретными фичами Bybit.

## Структура

```
gold-bot/
  data/                    # исторические свечи по каждой монете + top_symbols.txt
  logs/                    # логи живого бота (этап 4+)
  backtest/
    engine.py               # ПОРТФЕЛЬНЫЙ движок - общий баланс и лимиты на все монеты сразу
    strategies.py            # EMA trend-following (9/21, стоп по ATR) - одна и та же логика на каждую монету
    report.py                # печать отчёта (включая разбивку P&L по монетам)
    run_backtest.py          # запуск портфельного бэктеста
  bot/
    bybit_client.py           # обёртка над pybit: подключение, баланс, котировка
    top_symbols.py             # определяет топ-N монет по объёму за 24ч
    fetch_history_bybit.py    # выгрузка истории (один символ или TOP10, ключи не нужны)
    run_live.py                # заглушка живой торговли (этап 4, ещё не реализована)
  risk_gate.py               # ЖЁСТКИЕ риск-правила - единая точка проверки ордеров ПО ВСЕМУ ПОРТФЕЛЮ
  config.py                  # общие настройки (.env-переменные, НЕ риск-правила)
  .env.example               # шаблон API-ключей (реальный .env в git не коммитится)
  requirements.txt
```

## Риск-правила (зашиты в `risk_gate.py`, не в стратегии) — на весь портфель, не на монету

- Стартовый капитал (бэктест): **$50**, один общий на весь портфель
- Риск на сделку: **2%** от equity (потолок 3%, менять руками только в `risk_gate.py`)
- Стоп-лосс **обязателен** — без него `risk_gate.compute_position_size` бросает исключение, ордер не считается
- Эффективное плечо: максимум **5x** — **суммарно по всем открытым позициям во всех монетах**, а не по каждой монете отдельно (иначе при 10 одновременных позициях реальный риск мог бы дойти до 50x)
- Дневной лимит убытка: **-6%** от equity на начало дня, считается по **всему портфелю** -> при достижении останавливается торговля по ВСЕМ монетам до следующего дня
- **2 убыточные сделки за день** (по любым монетам вместе) -> тоже стоп по всему портфелю до следующего дня
- Минимальный ордер Bybit: **~5 USDT** — если позиция меньше, сделка пропускается
- Комиссия **taker 0.055%** и **funding rate** (приближение, см. `risk_gate.ASSUMED_FUNDING_RATE_PER_8H`) учтены в расчёте P&L для каждой монеты отдельно

И бэктест, и (в будущем) живой бот вызывают **один и тот же** `risk_gate.py` —
чтобы правила в тестах и в реальности не могли разойтись. Проверено юнит-тестами
на синтетических данных: портфельный лимит плеча ни разу не превышает 5x даже
при 5 одновременно открытых позициях в разных монетах.

## Стратегия: EMA trend-following (топ-N монет, M15)

Первая гипотеза для проверки, не финально настроенные параметры. Каждая
монета торгуется этой же логикой независимо:

- Быстрая EMA (9) пересекает медленную (21) -> вход по направлению пересечения
- Стоп-лосс: 2×ATR(14) от цены входа
- Выход: обратное пересечение EMA (разворот тренда) ИЛИ срабатывание стопа
- Крипта торгуется 24/7 — **нет** торгового окна и принудительного закрытия
  позиции к концу дня (в отличие от старой MT5-версии для золота)
- Список монет — топ-10 по объёму на Bybit на момент запуска `top_symbols.py`
  (обновляется, не фиксирован раз и навсегда)

## Этап 0: подключение

### 1. Установите зависимости

```powershell
cd gold-bot
python -m venv venv
venv\Scripts\python.exe -m pip install -r requirements.txt
```
(или активируйте venv через `venv\Scripts\activate`, если PowerShell разрешает запуск скриптов)

### 2. Как получить API-ключи Bybit

**Demo Trading (рекомендуется):**
1. Зарегистрируйтесь/войдите на [bybit.com](https://www.bybit.com).
2. В правом верхнем углу — переключатель **Demo Trading** (виртуальный баланс
   на реальных ценах mainnet). Включите его.
3. В настройках API (Account & Security -> API Management) создайте ключ
   именно в режиме Demo Trading — у него будут отдельные ключи от обычного
   аккаунта.
4. Скопируйте API Key и API Secret.

**Классический Testnet (альтернатива, менее реалистичный):**
1. Зарегистрируйтесь отдельно на [testnet.bybit.com](https://testnet.bybit.com) —
   это ДРУГОЙ аккаунт, не связанный с основным bybit.com.
2. Создайте API-ключ там же (API Management на testnet).

### 3. Настройте `.env`

```powershell
copy .env.example .env
```

Откройте `.env` в блокноте, впишите `BYBIT_API_KEY` и `BYBIT_API_SECRET`.
Для Demo Trading оставьте `BYBIT_DEMO=true`, `BYBIT_TESTNET=false` (значения
по умолчанию в `.env.example`). Для классического Testnet — наоборот,
`BYBIT_DEMO=false`, `BYBIT_TESTNET=true`.

### 4. Проверьте подключение

```powershell
venv\Scripts\python.exe bot\bybit_client.py
```

Должен вывести режим подключения, баланс USDT и текущую котировку BTCUSDT.

## Этап 1: данные

Публичные исторические данные — API-ключи НЕ нужны. Сначала определяем топ-10
монет по объёму, потом качаем историю по каждой:

```powershell
venv\Scripts\python.exe bot\top_symbols.py
venv\Scripts\python.exe bot\fetch_history_bybit.py TOP10
```

Первая команда сохранит список в `data\top_symbols.txt`, вторая скачает
`data\<SYMBOL>_M15.parquet` для каждой монеты из списка. Для честной проверки
стратегии нужно минимум 6 месяцев истории на каждую монету.

(Можно и по одной монете: `python bot\fetch_history_bybit.py ETHUSDT 15`.)

## Этап 3: бэктест

```powershell
venv\Scripts\python.exe backtest\run_backtest.py
```

Прогонит EMA-стратегию по всем монетам из `top_symbols.txt` с общим балансом
и общими риск-лимитами. Выведет честный отчёт (доходность, win rate, profit
factor, просадка, разбивка сырой сигнал/costы, P&L по каждой монете отдельно)
и сохранит `backtest\ema_trend_portfolio_trades.csv` +
`backtest\plots\ema_trend_portfolio_equity_curve.png`.

## Правила проекта (не нарушать)

- Работаем строго по этапам, не перепрыгиваем вперёд.
- Никогда не подключаемся к реальному счёту без явного письменного
  подтверждения в чате (`config.py -> IS_DEMO_ONLY`, `bot/bybit_client.py`
  печатает явное предупреждение, если `.env` указывает на live).
- Каждая позиция открывается только со стоп-лоссом, риск на сделку — через
  `risk_gate.py`, не напрямую в стратегии.
- Бэктест минимум 6 месяцев, прежде чем финализировать параметры стратегии.
